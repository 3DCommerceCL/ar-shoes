Pipeline de Entrenamiento
AR Shoe Try-On

De 100 fotos a un modelo de IA propio para detección de pies
Reemplaza el background subtraction con un modelo ONNX entrenado a medida

Python 3.10+ PyTorch SAM (Meta) ONNX Runtime Web ~$10 total 5–6 semanas

Visión General del Pipeline

100 fotos → SAM auto-etiqueta → 100,000 augmentadas
⬇
Entrenar FootSegNet (MobileNetV2 + U-Net) → Export ONNX ~5MB
⬇
Integrar en pose.js — sin cambios en el resto de la app
⬇ (aprendizaje continuo)
Users hacen try-on → captura automática → re-entrenar mensualmente

Fase 2 — Etiquetado con SAM (Meta)

SAM (Segment Anything Model) etiqueta cada foto con solo 1 clic en el pie. Tiempo estimado: 2–3 horas para las 100 fotos.

Instalación

pip install segment-anything opencv-python torch torchvision tqdm
# Descargar checkpoint SAM (342 MB):
wget https://dl.fbaipublicfiles.com/segment_anything/sam_vit_b_01ec64.pth

Uso del script

python 2_sam_label.py --checkpoint sam_vit_b_01ec64.pth
TeclaAcción
Clic izquierdo✓ Marcar píxeles de pie (incluir en máscara)
Clic derecho✗ Marcar fondo (excluir de máscara)
SGuardar máscara y pasar a la siguiente foto
RResetear clics en la imagen actual
QSalir y guardar progreso

Las máscaras se guardan en training/data/masks/ con el mismo nombre que la foto (PNG en blanco y negro: blanco = pie, negro = fondo).

Fase 4 — Entrenamiento del Modelo

Arquitectura: FootSegNet

ComponenteDetalle
EncoderMobileNetV2 pre-entrenado en ImageNet (pesos congelados primeras capas)
DecoderU-Net ligero de 5 niveles con skip connections
Entrada256 × 256 × 3 (RGB normalizado)
Salida256 × 256 × 1 (máscara de confianza 0–1)
PérdidaBCE + Dice Loss (robusto para clases desbalanceadas)
Parámetros~3.4M
Tamaño ONNX~5 MB

Instalación

pip install torch torchvision tqdm scikit-learn Pillow

Ejecutar entrenamiento

python 3_train_model.py --epochs 30 --batch_size 16

Integración en pose.js

El ONNX se integra reemplazando el background subtraction actual. No cambia nada en renderer.js ni app.js:

// pose.js — reemplazar initPose() y detectPose() con:
import { InferenceSession, Tensor } from 'onnxruntime-web';

let session = null;
export async function initPose() {
  session = await InferenceSession.create('./models/foot_segmenter.onnx');
}

export async function detectPose(videoEl) {
  // Preprocesar: video → Float32Array normalizado 1×3×256×256
  const tensor = preprocessVideo(videoEl);
  const result = await session.run({ input: tensor });
  const mask   = result.output.data;  // Float32Array 256×256
  return { data: mask, width: 256, height: 256 };
  // extractFootLandmarks() ya recibe este formato → sin cambios
}

Fase 6 — Aprendizaje Continuo

Cada sesión de try-on captura frames automáticamente en el navegador (IndexedDB). El usuario puede descargar esos frames mensualmente para re-entrenar el modelo.

1

Captura automática en producción

Cada 2 segundos durante una detección exitosa, se guarda un frame 256×256 + coordenadas del pie en IndexedDB del browser. Máximo 5,000 frames por dispositivo.

2

Exportar dataset mensualmente

El usuario presiona "Exportar dataset" → descarga un ZIP con todas las imágenes y un landmarks.json con las posiciones del pie.

3

Añadir al dataset y re-entrenar

Agregar las nuevas imágenes a data_augmented/. Fine-tuning rápido: python 3_train_model.py --epochs 10 (parte del modelo ya aprendió, converge más rápido).

4

Publicar nuevo modelo

Exportar ONNX → subir a GitHub Pages → los usuarios reciben el modelo mejorado automáticamente en su próxima visita.
